# 5.Dockerfile构建镜像

Dockerfile目录结构

基础镜像信息
维护者信息
镜像操作指令
容器启动时执行指令

指令

FROM- 镜像从那里来
	FROM java:8
MAINTAINER- 镜像维护者信息
	MAINTAINER hankuou2	
RUN- 构建镜像执行的命令，每一次RUN都会构建一层
CMD- 容器启动的命令，如果有多个则以最后一个为准，也可以为ENTRYPOINT提供参数
	CMD ["java", "-jar", "eureka-server-1.0.jar"]
VOLUME- 定义数据卷，如果没有定义则使用默认
USER- 指定后续执行的用户组和用户
WORKDIR- 切换当前执行的工作目录
	WORKDIR ${workdir}
HEALTHCHECH- 健康检测指令
ARG- 变量属性值，但不在容器内部起作用
EXPOSE- 暴露端口
	EXPOSE 8761 8762
ENV- 变量属性值，容器内部也会起作用
	ENV workdir=/zch/weifuwu/eureka-server  #工作目录
ADD- 添加文件，如果是压缩文件也解压

COPY- 添加文件，以复制的形式
	COPY . ${workdir}
ENTRYPOINT- 容器进入时执行的命令

构建镜像

cd到/root/zch/test目录下，创建Dockerfile文件

#基础镜像使用jdk1.8
FROM java:8
#作者
MAINTAINER hankuou2
# VOLUME /var/lib/docker/tmp并链接到容器的/tmp
VOLUME /tmp
# 将jar包添加到容器中并更名
ADD sangeng-blog.jar sangeng_blog.jar
# 运行jar包
RUN bash -c 'touch /sangeng_blog.jar'
ENTRYPOINT ["java","-jar","/sangeng_blog.jar"]
#暴露7777端口作为微服务
EXPOSE 7777

FROM openjdk:8
#工作目录
ENV workdir=/zch/weifuwu/eureka-server
COPY . ${workdir}
WORKDIR ${workdir}
EXPOSE 8761
CMD ["java", "-jar", "eureka-server-1.0.jar"]

docker build -t name .

# 1.docker run

docker run -id  \
-p 9999:3306 -p 9998:3307  \
--name=c_mysql  \
-v	/root/zch/conf:/etc/mysql/conf.d	\
-e MYSQL_ROOT_PASSWORD=123456 \
mysql:5.6 


-p	映射端口 9999是本机的端口  3306是容器的端口
可以映射多个端口
--name	容器的名字
-v	链接 本机的/root/zch/conf链接到容器里面的/etc/mysql/conf.d
可以链接多个目录
mysql:5.6	使用的镜像

# 2.Docker-compose

安装

curl -L https://get.daocloud.io/docker/compose/releases/download/v2.4.1/docker-compose-`uname -s`-`uname -m` > /usr/local/bin/docker-compose

将可执行权限应用于二进制文件：
sudo chmod +x /usr/local/bin/docker-compose
创建软链：
sudo ln -s /usr/local/bin/docker-compose /usr/bin/docker-compose
测试是否安装成功：
docker-compose versio

配置docker-compose.yml

#compose版本
version: "3" 

services:
  nginx:
    image: nginx:1.18.0
    ports:
      - "9999:8093"
    volumes:
      - /zch/nginx/html:/usr/share/nginx/html
      - /zch/nginx/logs:/var/log/nginx
      - /zch/nginx/conf:/etc/nginx

检查当前目录下compose.yml文件是否有语法错误

docker-compose config -q

启动所有docker-compose服务并后台运行

docker-compose up -d

# nginx

1.复制出nginx容器配置

mkdir /zch
cd /zch
# 1.运行容器
docker run -p 100:80 --name nginxxxx -d nginx:1.18.0
# 2.将容器内的配置文件拷贝到当前目录/app中:
docker container cp nginxxxx:/etc/nginx .
# 3.将文件nginx修改为conf 
mv nginx conf
# 4.创建文件夹nginx  这个文件夹会链接容器nginx
mkdir nginx
# 5.将conf目录拷贝到nginx目录
 cp -r conf nginx/
# 6.停止并删除容器
docker stop nginxxxx && docker rm nginxxxx

此时 nginx容器配置在/zch/nginx

2.配置docker-compose

5.重启容器

6.nginx实现反向代理

默认端口调到指定端口

实现www.zch1337.xyz直接跳转到www.zch1337.xyz:8093

我在修改时发现没实现反向代理
最后 百度知道了问题所在
在容器中 /etc/nginx/nginx.conf中有一方代码
include /etc/nginx/conf.d/*.conf 引入了默认的代码
去对应文件进行修改
vim /etc/nginx/conf.d/default.conf
server {
    listen 80;
    server_name localhost;
    location / {
        proxy_pass http://localhost:8093;
    }
}
成功!!!

# 解决1251错误

1、进入容器
docker exec -it mysql-test /bin/bash
2、进入mysql
mysql -u root -p
3、 对远程授权
GRANT ALL ON *.* TO 'root'@'%';
4、更改密码的加密规则
ALTER USER 'root'@'%' IDENTIFIED BY 'password' PASSWORD EXPIRE NEVER;
5、更改root的密码
ALTER USER 'root'@'%' IDENTIFIED WITH mysql_native_password BY '123456';
6、刷新数据库
flush privileges;
