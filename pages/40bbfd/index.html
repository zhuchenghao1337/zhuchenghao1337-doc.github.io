git

# 概念

个人理解：

分为本地工作区、暂存区、分支、远程代码库。

  • git add 命令：将本地工作区文件添加到暂存区。
  • git commit命令：将暂存区文件给某个分支一份，默认当前分支。
  • git push命令：将分支的数据上传到远程仓库。

# 命令

# 1.git init

git init 命令用于在目录中初始化仓库。

# 2.git clone

拷贝一份远程仓库。

# 3.git add

添加文件到暂存区。

# 4.分支管理

1.创建分支

git branch 名字

2.切换分支

git checkout 名字

3.分支重命名

git branch -M main

将当前分支的名字重名为main

# 5.git commit

git commit 命令将暂存区内容添加到本地仓库中。

git commit -m '备注信息'：默认提交到当前分支。

# 6.git push

将本地的分支版本上传到远程并合并。

格式:

git push <远程主机名> <本地分支名>:<远程分支名>

如果本地分支名与远程分支名相同，省略写法：

git push <远程主机名> <本地分支名>

eg：git push origin main

注意:

本机和远程有差异，可以强制推送 --force 参数

git push --force origin main

# 7.git reset

用于回退版本，可以指定退回某一次提交的版本。

git reset:暂存区回退到当前分支上一次commit的版本，工作区不变。

# 8.git pull

从远程获取代码并合并本地的版本。

格式：

git pull <远程主机名> <远程分支名>:<本地分支名>

eg:

git pull origin main:main

git pull origin main:main
# 一致省略
git pull origin main

将远程主机 origin 的 main 分支拉取过来，与本地的 main分支合并。

# 流程

# 配置

git config --global user.name "zch1337"
git config --global user.email "zhuchenghao1337@139.com"

# github

git init
git add .
git commit -m "first commit"
git branch -M main
git remote add origin https://github.com/zhuchenghao1337/dfa.git
git push -u origin main

# gitee

git init 
git add .
git commit -m "first commit"
git remote add origin https://gitee.com/zch1337/tetst.git
git push -u origin "master"

# bug问题

# 1.无法push？获取push失败

远程仓库和本地的代码不一致。

# 将远程仓库的拉下来并合并
git pull origin main
# 再次退送
git push -u origin main

# 强制推送 --force
git push --force origin main
