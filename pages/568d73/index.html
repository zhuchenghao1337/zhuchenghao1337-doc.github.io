# 一、初识Spring

# 1.OCP开闭原则

什么是OCP？

软件开发7大原则之一：开闭原则，是最核心，最基本的，其他6个原则都是为OCP服务的。

OCP核心是什么？

开？对扩展开放。

闭？对修改关闭。

只要在扩展系统功能的时候，不修改之前的代码，就符合OCP开闭原则。

为什么要OCP？

如果修改了之前的代码，那么所有的程序都要重新进行测试，非常麻烦。

# 2.DIP依赖倒置原则

什么是DIP？

面向接口编程，面向抽象编程，不要面向具体编程。

违背了DIP依赖倒置
private UserDao userDao = new UserDaoForMysql();
==> 程序员new了一个具体的对象，形成了 上 依赖 下（下面改动，上一层的代码也要改动，违背OCP原则）,在扩展的时候就非常麻烦。

遵守了DIP依赖倒置
private UserDao userDao;
==> 不用程序员new对象了，交给spring来干，就不会形成 上 依赖 下，下一层拓展了，这一层不要改动。

目的是什么？

降低程序的耦合度，提交扩展力。

# 3.IOC控制反转--编程思想

反转是什么

  1. 程序员 在程序中 不采用硬编码的方式 创建对象
  2. 程序员 在程序中 不采用硬编码的方式维护对象关系

**控制反转：**是一种思想，由于出现的比较新，没有纳入设计模式。

# 二、spring容器

# 依赖注入

spring通过依赖注入(DI)方法实现了IOC思想

  1. spring帮程序员new对象。
  2. spring帮程序员维护对象与对象的关系。

依赖注入有两种

  1. set注入(执行set方法给属性赋值)
  2. 构造方法注入(执行构造方法给属性赋值)

依赖是什么？注入是什么？

依赖：A对象和B对象的关系

注入：是一种手段，通过这种手段，可以让A和B产生关系

总结

spring通过依赖注入(DI)实现了IOC控制反转思想。

依赖：A与B的关系。

注入：一种手段，让A与B产生关系，包括：set注入和构架方法注入。

依赖注入：实现IOC的具体方法，通过注入的方法，让A与B两个对象产生关系。

# 四、Bean作用域-scope

spring在默认情况下，在根据配置文件new Spring上下文对象时，同时创建被管理的bean对象，这些bean对象是单例的。

可以在配置文件中，配置scope属性为prototype（默认为singleton），此时new 上下文对象时不会创建该对象，只有当调用getBean方法获取对象时才会创建对象，而且这个bean对象时多例的（每调用一次getBean方法就会新new一个对象）。

<bean id='***' class='***' scope='prototype'>

注意

scope其他值：

  1. singleton：默认，单例
  2. prototype：原型，多例
  3. request：一次请求对应一个bean
  4. session：一个会话对应一个bean
  5. 自定义scope
  6. ....
