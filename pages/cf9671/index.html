# 3. 常见注解

@Configuration 放到类上，表示这个类作为配置文件使用

@Bean 声明对象，把对象注入到容器中

@ImportResource 把xml中配置的对象引入到javaConfig中

@PropertySource() 读取属性配置文件 比如数据库配置文件

@SpringBootApplication

这是一个复合注解

包含@Configuration 被注解的类作为配置文件使用

包含@EnableAutoConfiguration 启用自动配置

包含@ComponentScan 找注解

# 5. 自动配置大揭秘 关键-原理

1.只要在pom.xml中引入redis依赖，就可以手工获取容器中的redis对象（redisTempla）
---这就是自动配置的其中一种 根据pom文件来配置相应的对象
2.条件注解是自动配置的关键
    ConditionalOnProperty：判断配置文件(yml文件)中如果有对应属性和值就初始化Bean
    ConditionalOnClass：环境中如果有对应字节码文件就初始化Bean(pom.xml中添加了redis的依赖，就会添加到字节码中) 
    ConditionalOnMissingBean：环境中如果没有对应Bean就初始化Bean

3.代码详解
@Configuration //表明这个类是一个配置类
public class UserConfig {
    @Bean //表明要创建一个对象
    @ConditionalOnclass({"redis.clients.jedis.Jedis"}) //条件注解 如果满足条件就创建对象
    //理解为：spring在字节码中去找redis这个id，如果找到了，创建对象（pom.xml中添加了redis的依赖，就会添加到字节码中）
    //创建对象成功后 就可以获取到对象了
    public User user(){
        return new User();
    }
    
}

为什么引入了starter-data-redis，我们就能项目中，直接拿redistemplate?

4.springboot大揭秘
    1.springboot 有来自父项目的依赖，里面写好了很多的条件注解 这是前提
    2.springboot的启动类 添加了@SpringBootApplication注解 这是一个复合注解
        其中：
            1.包含@Configuration 这个启动类作为配置类
            2.springboot中的autoconfig工程里把常用的对象的配置类都写好了
            3.包含@EnableAutoConfiguration 表示启用自动配置 启用之前写好的判断配置类
    3.所以 当pom.xml中添加了redis依赖，我们就可以拿到reids的对象redisTemplate对象
          当配置文件中配置了server.port的端口的信息，通过ConditionalOnProperty条件注解，springboot就帮配置并修改了server的对象信息

# 6.为什么springboot的内置服务器是tomcate？

引入了一个spring-web依赖

<dependency>
	<groupId>org.springframework.boot</groupId>
	<artifactId>spring-boot-starter-web</artifactId>
</dependency>

这里面写了内嵌了tomcat的依赖

由于springboot自动装配

就启动了tomcate服务器

# 7.切换springboot的内置服务器

1.排除Tomcat

<dependency>
            <groupId>org.springframework.boot</groupId>
            <artifactId>spring-boot-starter-web</artifactId>
            <!--排除tomcat依赖-->
            <exclusions>
                <exclusion>
                    <artifactId>spring-boot-starter-tomcat</artifactId>
                    <groupId>org.springframework.boot</groupId>
                </exclusion>
            </exclusions>
        </dependency>

2.添加其他的服务器依赖

<!--引入jetty的依赖-->
        <dependency>
            <artifactId>spring-boot-starter-jetty</artifactId>
            <groupId>org.springframework.boot</groupId>
        </dependency>

# 9. SpringBoot部署

SpringBoot 项目开发完毕后，支持两种方式部署到服务器：

①jar包(官方推荐) ===前后端分离项目

②war包

更改pom文件中的打包方式---默认为jar

改为war打包
&lt;packaging>war&lt;/packaging>

1.修改启动类

import org.springframework.boot.SpringApplication;
import org.springframework.boot.autoconfigure.SpringBootApplication;
import org.springframework.boot.builder.SpringApplicationBuilder;
import org.springframework.boot.web.servlet.support.SpringBootServletInitializer;

@SpringBootApplication
public class SpringbootDeployApplication extends SpringBootServletInitializer {

    public static void main(String[] args) {
        SpringApplication.run(SpringbootDeployApplication.class, args);
    }


    @Override
    protected SpringApplicationBuilder configure(SpringApplicationBuilder builder) {
        return builder.sources(SpringbootDeployApplication.class);
    }
}

2.指定打包的名称

   <build>
        <finalName>springboot</finalName>
        <plugins>
            <plugin>
                <groupId>org.springframework.boot</groupId>
                <artifactId>spring-boot-maven-plugin</artifactId>
            </plugin>
        </plugins>
    </build>
