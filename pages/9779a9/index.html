# 1.创建vuepress项目

注意：node.js >= 8.6

创建基本的项目
1.mkdir vuepress-starter && cd vuepress-starter
2.npm init
3.npm install -D vuepress
4.mkdir docs && echo '# Hello VuePress' > docs/README.md
5.package.json添加
    {
      "scripts": {
        "docs:dev": "vuepress dev docs",
        "docs:build": "vuepress build docs"
      }
    }
6.npm run docs:dev

创建自定义主题的项目
这是主题地址：https://vuepress-theme-reco.recoluan.com/views/1.x/installUse.html
1.安装
npm install vuepress-theme-reco --save-dev
2.引用
module.exports = {
  theme: 'reco'
}

# 4.1分类和标签

// .vuepress/config.js

module.exports = {
  theme: 'reco',
  themeConfig: {
     // 博客配置
    blogConfig: {
      category: {
        location: 2,     // 在导航栏菜单中所占的位置，默认2
        text: 'Category' // 默认文案 “分类”
      },
      tag: {
        location: 3,     // 在导航栏菜单中所占的位置，默认3
        text: 'Tag'      // 默认文案 “标签”
      },
      socialLinks: [     // 信息栏展示社交信息
        { icon: 'reco-github', link: 'https://github.com/recoluan' },
        { icon: 'reco-npm', link: 'https://www.npmjs.com/~reco_luan' }
      ]
    }
  }
}

# 4.4评论

如果你想默认不加载评论，而只在某些页面显示评论功能，可以在 valineConfig 或 vssueConfig 中设置 showComment: false，并在需要展示评论的页面 设置 isShowComments: true

如果仅是某篇文章不想设置开启评论功能，可以在 front-matter 设置 isShowComments: false

# 4.6侧边栏

主题已经将某篇文章的侧边栏抽离出来放在右边，被称为「子侧边栏」

左边的侧边栏的仅会在自定义侧边栏时显示，它所显示的知识文章的分类和目录，我们称它为「原侧边栏」

  • 对于「原侧边栏」，只能通过自定义侧边栏实现，无法通过Front Matter中的sidebar与subSidebar配置显示与否。
  • 对于「子侧边栏」，可通过配置config.js配置全局开启与关闭，也可以通过Front Matter中的subSidebar定义每篇文章中的开启与关闭。
子侧边栏是否打开
// .vuepress/config.js
module.exports = {
  themeConfig: {
    subSidebar: 'auto'//在所有页面中启用自动生成子侧边栏，原 sidebar 仍然兼容
  }
}

---
subSidebar: false
---

2.写部署脚本

采用2个项目来部署，第一个项目为私密项目，放源项目；

第二个项目公开，放生成的dist文件

一键部署脚本
作用：build根项目，将生成的dist复制到blog，然后将blog提交

name: deploy.sh

#!/usr/bin/env sh
# 确保脚本抛出遇到的错误
set -e
# 生成静态文件
npm run build
# 清除之前的
rm -rf ../blog/*
# 使用最新的
cp -rf docs/.vuepress/dist/* ../blog/
# 进入生成的文件夹
cd ../blog
# 如果是发布到自定义域名
# echo 'www.example.com' > CNAME

git init
git add -A
git commit -m 'deploy'
git branch -M main
git remote add origin https://github.com/zhuchenghao1337/zhuchenghao1337.github.io.git
